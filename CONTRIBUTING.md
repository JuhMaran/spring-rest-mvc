# Contributing Guide

Obrigado por considerar contribuir com este projeto!  
Este repositório faz parte de estudos e projetos pessoais, então toda ajuda é bem-vinda. 🚀

## 1. Diretrizes de Colaboração

- Este projeto aceita contribuições de forma aberta, desde que sigam as boas práticas descritas aqui.  
- Todas as mudanças devem ser feitas via **Pull Request (PR)**  
- A branch `main/master` é **protegida** e não aceita commits diretos.  
- Mantenha um comportamento respeitoso e colaborativo.

## 2. Reportando Bugs

Se você encontrou um erro:

1. Verifique se já existe uma issue semelhante aberta.
2. Caso não exista, crie uma nova issue contendo:
   - Descrição clara do problema
   - Passos para reproduzir
   - Comportamento esperado vs. atual
   - Prints ou logs (se possível)

## 3. Sugestão de Melhorias

Para sugerir melhorias:

- Abra uma issue com a tag `enhancement`
- Explique:
   - O problema que a melhoria resolve
   - A proposta de solução

## 4. Padrões de Código

- Escreva código limpo e legível
- Use nomes de variáveis descritivos
- Siga o padrão já existente no projeto
- Comente trechos importantes quando necessário
- Evite código desnecessariamente complexo

## 5. Fluxo de Trabalho (Workflow)

### 5.1. Faça um Fork

Crie uma cópia do repositório na sua conta.

### 5.2. Clone o repositório

```bash
git clone https://github.com/seu-usuario/nome-do-repo.git
```

### 5.3. Crie uma branch

```bash
git checkout -b minha-feature
```

### 5.4. Faça suas alterações

Edite os arquivos necessários.

### 5.5. Commit

```bash
git add .

git commit -m "feat: adiciona nova funcionalidade"
```

### 5.6. Envie para o GitHub

```bash
git push origin minha-feature
```

### 5.7. Abra um Pull Request

- Explique claramente o que foi feito
- Relacione com alguma issue, se houver

## 6. Outras Formas de Contribuir

Você também pode contribuir de outras formas:

- Melhorando documentação (README, comentários, etc.)
- Corrigindo erros de digitação
- Ajudando a responder issues
- Revisando Pull Requests

## 7. Como Adicionar Arquivos

### 7.1. Via Navegador

- Clique em "Add file"
- Selecione "Upload files"
- Envie seus arquivos
- Faça commit direto ou via branch

### 7.2. Via Linha de Comando

```bash
git add . 

git commit -m "chore: adiciona novos arquivos" 

git push
```

## 8. Observações

- Este repositório é voltado para estudos e projetos pessoais
- Não há vínculo com projetos comerciais ou empresariais
- Algumas implementações podem seguir fins educacionais

## 9. Dúvidas

Se tiver dúvidas, sinta-se à vontade para abrir uma issue 🙂

_Obrigada por contribuir!_ ❤️
